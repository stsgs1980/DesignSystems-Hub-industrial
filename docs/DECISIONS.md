## 1. Навигация "Компоненты"

### Задача
При нажатии на карточку "Компоненты" пользователь должен "провалиться" внутрь, а не видеть всплывающее окно.

### Варианты реализации

#### Вариант 1: Full-page Transition ✅ ПРИНЯТ
```
Главная страница → Анимация → Полноэкранный view компонентов
                                    ↓
                              Кнопка "НАЗАД" → Возврат
```

**Преимущества:**
- Ощущение "погружения" и глубины
- Timeline Navigator остаётся видимым
- Больше места для контента
- Можно добавить breadcrumbs

**Реализация:**
- State `componentsViewOpen: boolean`
- Framer Motion AnimatePresence для анимации
- Slide/fade переход
- Кнопка "НАЗАД" в header
- Timeline показывает специальную позицию или скрывается

**Технические детали:**
```tsx
const [componentsViewOpen, setComponentsViewOpen] = useState(false)

// Анимация
<AnimatePresence mode="wait">
  {componentsViewOpen ? (
    <motion.div
      key="components"
      initial={{ opacity: 0, x: 100 }}
      animate={{ opacity: 1, x: 0 }}
      exit={{ opacity: 0, x: -100 }}
    >
      {/* Components View */}
    </motion.div>
  ) : (
    <motion.div key="main">
      {/* Main Page */}
    </motion.div>
  )}
</AnimatePresence>
```

#### Вариант 2: Sheet (выезжающая панель)
```
         ┌─────────────────────────────┐
         │                             │
   ←←←←←←│   КОМПОНЕНТЫ                │
         │                             │
         │   50+ компонентов...        │
         │                             │
         └─────────────────────────────┘
```

**Преимущества:**
- Быстрый доступ
- Контекст главной страницы не теряется
- Стандартный паттерн для мобильных

**Недостатки:**
- Меньше места на мобильных
- Не даёт ощущения "проваливания"
- Sheet из shadcn/ui нужно подключать

---

#### Вариант 3: Секция внизу страницы
```
...главная страница...
│
▼
┌─────────────────────────┐
│   КОМПОНЕНТЫ            │
│   Полная секция         │
│   50+ компонентов       │
└─────────────────────────┘
```

**Преимущества:**
- Простая реализация
- Timeline автоматически работает
- Единый скролл

**Недостатки:**
- Нет ощущения "отдельного пространства"
- Главный экран перегружается
- Не "проваливание", а просто навигация

---

### Принятое решение
**Вариант 1: Full-page Transition**

**Дата:** 2025-01-XX
**Причина:** Лучшее погружение, больше места, современный UX

## 2. Timeline Navigator

### Задача
Вертикальная навигация слева с маркером текущей позиции.

### Реализация ✅ ЗАВЕРШЕНО
- IntersectionObserver для определения секции
- Framer Motion для анимации маркера
- Fixed позиция слева
- Tooltip при наведении
- Smooth scroll при клике

**Дата:** 2025-01-XX

## 3. Dark/Light Theme

### Задача
Переключение между тёмной и светлой темой.

### Реализация ✅ ЗАВЕРШЕНО
- ThemeProvider (next-themes)
- ThemeToggle компонент в header
- CSS переменные в globals.css
- Смягчённая тёмная тема с холодным оттенком

**Дата:** 2025-01-XX
